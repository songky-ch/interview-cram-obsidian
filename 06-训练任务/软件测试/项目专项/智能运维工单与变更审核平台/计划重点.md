## 项目定位

智能运维工单与变更审核平台是一个综合类测试项目, 主线为:

告警/日志接入 -> AI Agent 初筛 -> 工单创建 -> 变更申请 -> 多级审核 -> 执行/回滚 -> 验收关闭 -> 知识库沉淀

项目用于串联简历中的 AI Agent、现网日志分析、平台测试、接口自动化、性能测试、移动端自动化、IoT 链路定位、数据库校验和 Linux 日志分析经验。

## 一周训练重点

| 天数 | 主题 | 训练目标 |
|---|---|---|
| Day 1 | 项目业务与测试流程 | 讲清项目背景、业务主线、角色、风险和测试阶段 |
| Day 2 | 功能测试与复杂审核流程 | 掌握状态流转、角色权限、异常分支和数据一致性讲法 |
| Day 3 | 接口测试与自动化 | 讲清 Postman、Python、Pytest、数据库断言和 Jenkins 回归 |
| Day 4 | AI Agent 测试专项 | 讲清 Prompt、知识库、多轮对话、工具调用和回归评测 |
| Day 5 | 性能测试与日志定位 | 讲清 Locust 场景设计、指标分析、Linux 和数据库定位 |
| Day 6 | Web/APP/IoT 自动化 | 讲清 Appium、WebDriverAgent、UIAutomator、Wireshark、串口工具的项目应用 |
| Day 7 | 安全测试与综合答疑 | 讲清越权、流程绕过、提示词注入、上线验收和综合项目介绍 |

## 技术栈映射

- AI Agent: 回答准确性、上下文一致性、知识库命中、工具调用、异常输入、历史样本回归。
- Python + Pytest: 接口自动化、参数化、fixture、数据库断言、主流程回归。
- Postman: 接口联调、鉴权验证、异常参数构造、缺陷复现。
- Locust: 工单查询、审批提交、Agent 问答、知识库检索等接口性能测试。
- Jenkins + Git: 自动化脚本版本管理、定时回归、失败用例追踪。
- Linux + 日志分析: 查看服务日志、异常栈、资源占用和接口超时原因。
- MySQL/PostgreSQL: 校验工单状态、审批记录、执行记录、知识库数据。
- Appium/WebDriverAgent/UIAutomator: 移动端审批、工单查看、消息提醒等核心流程自动化。
- Wireshark/抓包/串口: 分析设备告警、日志上报、设备侧异常和服务端接入问题。
- 安全测试: 鉴权、越权、审批绕过、重放请求、敏感信息泄露、提示词注入。
