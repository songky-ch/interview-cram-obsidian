## 面试主题

Day1: 测试流程、测试策略、缺陷闭环

## 项目范围

- 睿思 AI Agent(数字员工): 现网日志分析、分析件初筛、业务问答、多轮对话、工具调用。
- 生产维护数据分析平台 / 现网维护数据分析平台: 数据流转、权限控制、报表展示、多模块联动、接口测试、版本验收。

## 高频问题

### 1. AI Agent 新需求如何做测试分析和测试策略设计?

考察点:

- 是否能区分 AI Agent 测试和传统 Web / App 测试。
- 是否能围绕准确性、幻觉、上下文一致性、工具调用、异常输入设计测试策略。
- 是否能结合睿思 AI Agent 项目场景回答。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#1. AI Agent 新需求如何做测试分析和测试策略设计]]

### 2. AI Agent 回归测试数据集如何设计?

考察点:

- 是否能设计正向、负向、边界、历史缺陷、工具调用、多轮对话样本。
- 是否意识到 AI Agent 回归数据集需要标准答案、关键要点、禁止项和评估规则。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#2. AI Agent 回归测试数据集如何设计]]

### 3. Agent 输出不固定时, 如何判断用例通过或失败?

考察点:

- 是否知道不能做完全字符串匹配。
- 是否能按业务结论、关键事实、依据可追溯、禁止项、工具调用预期判断。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#3. Agent 输出不固定时如何判断用例通过或失败]]

### 4. 只改 Prompt 时, 是否需要完整回归?

考察点:

- 是否能按 Prompt 改动影响面分层回归。
- 是否能覆盖任务目标、上下文、约束、输出格式、工具调用说明等影响点。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#4. 只改 Prompt 时是否需要完整回归]]

### 5. 平台需求涉及数据流转、权限控制、报表展示和多模块联动时, 如何设计测试方案?

考察点:

- 是否能按端到端业务链路拆解测试范围。
- 是否覆盖功能、权限、接口、数据库、报表统计、多模块联动和回归自动化。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#5. 平台复杂需求如何设计测试方案]]

### 6. 页面报表展示数据和接口返回数据不一致, 如何定位?

考察点:

- 是否能按页面、请求参数、接口响应、前端渲染、数据库、统计口径、缓存、权限范围定位。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#6. 页面报表和接口返回不一致如何定位]]

### 7. 怀疑是 MySQL 统计口径问题时, 如何验证?

考察点:

- 是否能统一筛选条件。
- 是否能用 count、group by、sum / avg、join、distinct、权限范围和数据时效性拆解验证。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#7. MySQL 如何验证统计口径问题]]

### 8. 严重缺陷从提交到修复上线, 如何跟进生命周期?

考察点:

- 是否能完整覆盖缺陷记录、分级、影响范围、提交、修复跟进、回归验证、上线验收和知识沉淀。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#8. 严重缺陷如何推动生命周期闭环]]

### 9. 开发认为不是 bug, 但测试认为影响质量, 如何处理?

考察点:

- 是否能用需求、验收标准和用户影响做证据化沟通。
- 是否能推动产品、开发、测试共同做风险决策。

优秀答案见: [[04-优秀答案/软件测试/Day1-测试流程测试策略缺陷闭环优秀答案#9. 开发认为不是 bug 时如何处理]]

### 10. 如何完整讲述一次项目测试流程?

考察点:

- 是否能按项目背景、需求分析、测试方案、测试执行、缺陷闭环、回归验收、结果沉淀组织回答。
